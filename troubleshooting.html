Troubleshooting

Common issues and how to fix them.

Installation issues

Network not working during install

The installer attempts to configure networking automatically — ethernet first, then Wi-Fi. If it fails:

  1. Switch to tty2 with Ctrl+Alt+F2
  2. Check available interfaces: ip link
  3. Scan for Wi-Fi: nmcli dev wifi list
  4. Connect manually: nmcli dev wifi connect SSID password PASSWORD
  5. Switch back to tty1 (Ctrl+Alt+F1) and re-run: /root/install.sh

Installer fails at pacstrap

If pacman-key isn't initialized, the installer runs pacman-key --init && pacman-key --populate archlinux automatically. If it still fails, ensure you have a working internet connection and try again.

"Failed to switch root" on boot

This usually means the initramfs can't find the root filesystem. Make sure:

  • The archiso hooks are present in mkinitcpio.conf
  • The initramfs is rebuilt after changes

Post-installation issues

SDDM doesn't auto-login

Check that /etc/sddm.conf.d/autologin.conf exists and contains:

[Autologin]
User=selector-user
Session=plasmatv-selector.desktop
Relogin=true

Also verify /usr/share/wayland-sessions/plasmatv-selector.desktop exists.

Electron apps don't launch

Ensure the electron package is installed: sudo pacman -S electron. The installer includes it in the package list, but if you used the transform script on an existing Arch install, you may need to install it manually.

No audio

PlasmaTV uses PipeWire. Check that the service is running:

systemctl --user status pipewire
systemctl --user status wireplumber

If not running, start them manually:

systemctl --user enable --now pipewire pipewire-pulse wireplumber

Switching between Bigscreen and Desktop doesn't work

The switch scripts update the Session= line in /etc/sddm.conf.d/autologin.conf and restart SDDM. If it fails, check:

  • The session file name in /usr/share/wayland-sessions/ matches what the switch scripts expect
  • SDDM is running: systemctl status sddm

Building issues

mkarchiso fails with permission errors

Previous build artifacts may have root-owned files. Clean them:

sudo rm -rf work out

Package not found during build

Ensure the package name is correct and available in the Arch repos. For AUR packages, you'll need to handle them separately — the build only includes official repo packages.

Getting help

Open an issue on GitHub with details about the problem and any relevant log output.
